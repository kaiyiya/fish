# 智能鱼类电商推荐系统 - 项目功能分析报告

## 一、系统架构分析

### 1.1 系统端分析

本系统采用**前后端分离**的架构设计，包含以下端：

#### 前端端（2个端）
1. **微信小程序端**
   - 基于 Taro 3.x 框架开发
   - 支持微信小程序原生功能
   - 可编译为微信小程序代码

2. **H5端（Web端）**
   - 同样基于 Taro 3.x 框架
   - 一套代码多端运行
   - 支持浏览器访问

#### 后端端（1个端）
1. **RESTful API 服务端**
   - 基于 NestJS 10.x 框架
   - 提供统一的 API 接口
   - 支持多端调用

#### 数据存储端
1. **MySQL 数据库**
   - 存储业务数据
   - 15+ 张数据表
   - 支持事务处理

2. **Redis（可选）**
   - 缓存推荐结果
   - 提升系统性能

#### AI服务端
1. **深度学习模型服务**
   - TensorFlow 模型推理
   - 图像识别服务
   - 推荐算法服务

### 2.5 智能推荐模块（Recommendation Module）

**后端功能：**
- ✅ 个性化推荐（GET /ai/recommend）
- ✅ 多种推荐策略：
  - 基于用户的协同过滤（UserCF）
  - 基于物品的协同过滤（ItemCF）
  - 基于内容的推荐（Content-Based）
  - 热门推荐（Popularity-Based）
  - 混合推荐（Hybrid）
- ✅ 推荐策略智能选择
- ✅ 推荐日志记录
- ✅ 推荐效果评估

**前端功能：**
- ✅ 推荐商品展示（首页）
- ✅ 推荐理由展示
- ✅ 推荐商品点击追踪

**数据库表：**
- `recommendation_log` 表：推荐日志
- `user_behavior` 表：用户行为数据

**技术亮点：**
- 多种推荐算法实现
- 协同过滤算法（UserCF、ItemCF）
- 相似度计算（余弦相似度、皮尔逊相关系数）
- 冷启动问题处理
- 推荐结果缓存

**算法实现：**
- UserCF：时间复杂度 O(n²×m)
- ItemCF：时间复杂度 O(m²×n)
- 混合推荐：加权融合策略
- 行为权重设计（浏览1.0、收藏3.0、加购5.0、购买10.0）
